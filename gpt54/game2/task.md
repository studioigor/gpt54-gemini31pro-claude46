# «Бумажный Кораблик» — Концепт-план

## 1. ВЫСОКОУРОВНЕВАЯ ИДЕЯ

**Жанр:** Аркадное путешествие / раннер с вертикальным скроллом (сверху вниз)

**Концепт:** Маленький бумажный оригами-кораблик плывёт по дождевому ручью, который начинается как тонкая струйка воды на тротуаре и постепенно превращается в бурную реку. Мир нарисован акварелью — видны мазки кисти, краска растекается, бумага мокнет. Кораблик хрупкий, но по пути он собирает оригами-спутников (журавлей, рыбок, лягушек), которые дают ему силы.

**Почему цепляет:**
- Эмоциональный контраст: крошечный хрупкий кораблик vs. нарастающая стихия
- Визуальный стиль, который невозможно спутать ни с чем
- Интуитивное управление (двигай кораблик, уворачивайся, собирай)
- Прогрессия от спокойствия к хаосу создаёт естественное напряжение

---

## 2. ВИЗУАЛЬНЫЙ СТИЛЬ

**Палитра:** Тёплые акварельные тона — охра, сиена, индиго, изумруд.
Вода — полупрозрачный ультрамарин поверх текстуры бумаги.

**Фон:** Статичная текстура акварельной бумаги (grain visible).
Поверх неё — всё рисуется с эффектом "мокрой акварели":
- Мягкие края у объектов (blur + transparency)
- Подтёки краски на границах воды и берегов
- Кляксы как декоративные элементы

**Как достигается без движка:**
- Canvas 2D — основной рендер
- Текстура бумаги — фоновое изображение (можно сгенерировать процедурно через noise)
- Акварельные объекты — спрайты с мягкими alpha-краями
- Подтёки и кляксы — процедурные: наложение полупрозрачных кругов разного размера с gaussian blur через shadow
- Рябь воды — синусоидальная деформация + анимированные полупрозрачные эллипсы

**Технический приём для "акварельности":**
```
Каждый объект рисуется в 3 слоя:
1. Размытая тень (цвет объекта, 20% opacity, offset 2-3px) — "растеклась краска"
2. Основная форма (70% opacity) — сам объект
3. Яркое ядро (90% opacity, чуть меньше) — "густая краска в центре"
→ Это создаёт эффект акварельного мазка
```

## 3. ИГРОВЫЕ ОБЪЕКТЫ

### 3.1 Кораблик (игрок)
- Оригами-кораблик, вид сверху (ромбовидная форма с "парусом")
- Рисуется линиями-складками поверх белого треугольника
- Слегка покачивается на воде (rotation ±3° по синусоиде)
- При намокании (получение урона) — белый цвет кораблика темнеет, складки расползаются
- Состояние: 3 уровня "сухости" (белый → серый → полупрозрачный → game over)

### 3.2 Ручей (игровое поле)
- Вертикальная полоса воды, скроллящаяся сверху вниз
- Берега — неровные акварельные края (процедурная генерация зигзагом + сглаживание)
- Ширина и скорость ручья меняются по ходу игры:
  - Начало: узкий (40% экрана), медленный
  - Середина: средний (60%), быстрее
  - Финал: почти весь экран, стремительный

### 3.3 Препятствия

**Камушки** — округлые акварельные серо-коричневые формы.
Статичные. Столкновение отбрасывает кораблик и отнимает 1 уровень сухости.

**Чернильные кляксы** — тёмно-фиолетовые пятна, медленно расползающиеся.
Анимированы: пульсируют, увеличиваясь и уменьшаясь.
При касании — мгновенно -1 сухость + экран на секунду "пачкается" чернилами.

**Водовороты** — спиральная анимация из полупрозрачных голубых дуг.
Затягивают кораблик к центру (гравитация). Не наносят урон напрямую,
но могут затянуть в камень или кляксу.

**Листья** — не наносят урон, но блокируют обзор.
Большой акварельный лист плывёт поверх кораблика, скрывая его на 2-3 секунды.

### 3.4 Собираемые предметы

**Оригами-журавлик** — восстанавливает 1 уровень сухости.
Визуально: белая бумажная птичка, машет крыльями (2 кадра анимации).

**Оригами-звёздочка** — очки (×10 за штуку).
Маленькая, золотисто-жёлтая, вращается.

**Оригами-лягушка** (редкая) — даёт щит на 5 секунд.
Кораблик окружается зелёным пузырём. Одно бесплатное столкновение.

## 4. СТРУКТУРА УРОВНЕЙ (БЕСШОВНАЯ)

Игра — один непрерывный поток. Но внутренне разбита на фазы:

### Фаза 1 — "Тротуар" (0-30 секунд)
- Фон: бежевая бумага, текстура асфальта акварелью
- Берега: прямые линии бордюра
- Препятствия: только камушки, редко
- Музыка: тихое пиано, звук капель дождя
- Скорость: низкая
- Назначение: обучение без текста — игрок понимает управление

### Фаза 2 — "Канавка" (30-90 секунд)
- Фон: коричнево-зелёный, появляется трава по краям
- Берега: неровные, ручей виляет
- Препятствия: камушки + первые кляксы
- Появляются журавлики и звёздочки
- Скорость: средняя
- Ручей шире, течение видно по анимированным линиям

### Фаза 3 — "Парковый ручей" (90-180 секунд)
- Фон: зелёный, цветочки по берегам (акварельные пятна розового/жёлтого)
- Водовороты появляются
- Листья начинают падать сверху
- Музыка: добавляются струнные
- Скорость: выше средней

### Фаза 4 — "Река" (180+ секунд)
- Фон: глубокий синий, берега далеко
- Все типы препятствий, высокая плотность
- Появляется дождь (частицы — косые линии поверх всего)
- Экран периодически "вздрагивает" от грома (screen shake)
- Скорость: высокая и продолжает расти
- Лягушки появляются чаще (баланс сложности)
- Бесконечная фаза — играешь до game over

**Переходы между фазами:**
Плавное изменение палитры за 10 секунд (lerp цветов фона и берегов).
Никаких экранов загрузки или надписей "Level 2".

## 5. УПРАВЛЕНИЕ

**Desktop:** мышь — кораблик стремится к позиции курсора
(плавное следование с инерцией, не телепортация).
Альтернатива: A/D или стрелки для горизонтального движения.

**Mobile:** касание/тап — кораблик движется к точке касания.
Или наклон устройства (DeviceOrientation API) — опционально.

**Важно:** Кораблик движется ТОЛЬКО горизонтально (влево-вправо).
Вертикальная позиция фиксирована (~70% от верха экрана).
Мир скроллится сверху вниз — иллюзия движения вперёд.

---

## 6. СИСТЕМА ОЧКОВ И ПРОГРЕССА

**Очки:**
- Время в секундах × 1 (базовый счёт за выживание)
- Звёздочки: +10
- Нет комбо-системы — простота важнее

**Рекорд:** localStorage — лучший результат.
При побитии рекорда — экран game over "подсвечивается" золотым.

**Game Over экран:**
Кораблик "размокает" — анимация: белый цвет растворяется в воде,
складки расходятся, бумага становится прозрачной и исчезает.
Пауза 1.5 сек → появляется надпись акварельным шрифтом:
"Кораблик проплыл [X] метров" и кнопка "Ещё раз" (нарисованная
как бумажная кнопка с загнутым уголком).

## 7. ЗВУКОВОЕ ОФОРМЛЕНИЕ

**Библиотека:** Tone.js (доступна через CDN)

**Эмбиент:**
- Звук воды — белый шум через lowpass filter, лёгкая модуляция
- Капли дождя — рандомные короткие высокие ноты (pluck synth)
- Интенсивность дождя растёт с фазами

**Музыка:**
- Простая мелодия — пиано, 4-6 нот, зацикленная
- С каждой фазой добавляется слой (strings → bass → percussion)
- Реализация: Tone.js Sequence + несколько синтезаторов

**SFX:**
- Подбор звёздочки: высокий "дзинь" (FM synth)
- Подбор журавлика: мягкий аккорд
- Столкновение: глухой "буп" + вибрация (navigator.vibrate на мобильных)
- Водоворот рядом: низкий гул

## 8. ТЕХНИЧЕСКАЯ АРХИТЕКТУРА

```
index.html — единственный файл
├── <style> — минимальный CSS (canvas fullscreen, game over overlay)
├── <canvas> — основной игровой холст
└── <script>
    ├── Config — константы (скорости, размеры, тайминги фаз)
    ├── Utils — математика (lerp, clamp, distance, noise)
    ├── WatercolorRenderer — отрисовка "акварельных" форм
    │   ├── drawWatercolorCircle(x, y, r, color) — многослойный круг
    │   ├── drawWatercolorShape(points, color) — произвольная форма
    │   └── drawPaperTexture() — фоновый шум
    ├── River — генерация берегов, скроллинг, расширение
    ├── Boat — позиция, состояние сухости, рендер, покачивание
    ├── ObstacleManager — спавн/удаление/обновление препятствий
    │   ├── Rock, InkBlot, Whirlpool, Leaf — типы
    │   └── spawnTable[] — таблица вероятностей по фазам
    ├── CollectibleManager — спавн/удаление бонусов
    │   ├── Crane, Star, Frog — типы
    │   └── spawnTable[]
    ├── ParticleSystem — дождь, брызги, растворение кораблика
    ├── AudioManager — Tone.js обёртка
    │   ├── initAmbient(), initMusic()
    │   ├── setPhase(n) — наращивание слоёв
    │   └── playSFX(type)
    ├── PhaseManager — отслеживание времени, переключение фаз
    ├── GameState — очки, рекорд, пауза, game over
    └── GameLoop — requestAnimationFrame, delta time
        ├── update(dt)
        └── render(ctx)
```

**Размер:** ~250-350 строк JS + 30 строк CSS + 20 строк HTML

**Внешние зависимости:**
- Tone.js (CDN) — звук
- Никаких фреймворков, чистый Canvas 2D

## 9. ПРОЦЕДУРНАЯ ГЕНЕРАЦИЯ

**Берега ручья:**
```
Алгоритм:
1. Базовая ширина W(t) — растёт с фазой
2. Центр ручья Cx(t) — Perlin noise для плавного виляния
3. Левый берег: Cx - W/2 + noise(y, seed1) × 20
4. Правый берег: Cx + W/2 + noise(y, seed2) × 20
5. Отрисовка: заливка между берегами полупрозрачным голубым
6. Берега: толстая "рваная" линия с shadow для акварельности
```

**Размещение объектов:**
```
Каждые N пикселей скролла (N зависит от фазы):
1. Бросок: препятствие или бонус? (70/30)
2. Выбор типа по spawnTable текущей фазы
3. Позиция X: случайная в пределах ручья
4. Проверка: не слишком ли близко к другим объектам (min distance)
5. Создание объекта
```

## 10. КЛЮЧЕВЫЕ ВИЗУАЛЬНЫЕ ЭФФЕКТЫ

**Рябь воды:**
Массив из ~20 эллипсов, полупрозрачных, движущихся вниз.
Каждый: x, y, radiusX, radiusY, alpha (затухает со временем).
Новые появляются рандомно. Рисуются поверх воды, под объектами.

**Дождь (фаза 4):**
Массив частиц (100-200 шт.). Каждая — короткая белая линия
под углом ~75°, движется вниз-влево. Alpha 30-50%.

**Экран-тряска (гром):**
Раз в 10-20 секунд в фазе 4: canvas translate на рандомный offset (±3px)
на 300ms, затем возврат. Одновременно — белая вспышка (overlay 10% opacity).

**Растворение кораблика (game over):**
Заменяем кораблик на 30-40 частиц белого цвета.
Частицы разлетаются + затухают (alpha → 0) за 1.5 секунды.

## 11. АДАПТИВНОСТЬ

- Canvas масштабируется под window, сохраняя пропорции
- Все координаты в % от ширины/высоты canvas
- Touch-события для мобильных
- Минимальный размер: 320×480 (вертикальная ориентация)
- На десктопе: ограничение ширины canvas (max 500px),
  центрирование, бумажная текстура по бокам как "рамка"
